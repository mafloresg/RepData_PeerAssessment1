Reproducible Research: Peer Assessment 1

Introduction

It is now possible to collect a large amount of data about personal movement using activity monitoring devices such as a Fitbit, Nike Fuelband, or Jawbone Up. These type of devices are part of the quantified self movement a group of enthusiasts who take measurements about themselves regularly to improve their health, to find patterns in their behavior, or because they are tech geeks. But these data remain under-utilized both because the raw data are hard to obtain and there is a lack of statistical methods and software for processing and interpreting the data.

This assignment makes use of data from a personal activity monitoring device. This device collects data at 5 minute intervals through out the day. The data consists of two months of data from an anonymous individual collected during the months of October and November, 2012 and include the number of steps taken in 5 minute intervals each day.



Data

The data for this assignment can be downloaded from the course web site:

  • Dataset: Activity monitoring data [52K]

The variables included in this dataset are:

  • steps: Number of steps taking in a 5-minute interval (missing values are coded as NA)
  • date: The date on which the measurement was taken in YYYY-MM-DD format
  • interval: Identifier for the 5-minute interval in which measurement was taken

The dataset is stored in a comma-separated-value (CSV) file and there are a total of 17,568 observations in this dataset.

What is the average daily activity pattern?

1. Make a time series plot of the 5-minute interval (x-axis) and the average number of steps taken, averaged across all days (y-axis)

I use the aggregate() function to calculate the mean of the steps, grouping by interval. The result is saved into avgStepsByInterval. Then, I change the names of the columns of this variable into a meaningful ones. Third, I use the Base plotting system plot function to show the mean of steps taken in each 5-minute interval.

avgStepsByInterval <- aggregate(activityData$steps, by=activityData["interval"], FUN=mean, na.rm = TRUE);
names(avgStepsByInterval) <- c("interval","avgsteps");

Its interesting to see, as was pointed in the course forum (see this post), that if the intervals are used as the x-axis, the plot system assumes that there are missing values. This is because the intervals are used as numbers, but instead they are the times of the day. So, for the plotting system there is a kind of gap between the intervals 55 and 100, 155 and 200, etc, when there is no such a gap, as 55 means 0:55am, 100 is 1:00am, etc. The plotting system treat the interval like a numeric value and fill the gaps, plotting weird intervals like 1:70am and so.

In order to avoid this, some functions are used in the next chunk code. First, inside the plot function, the values for the x axis are changed to be dates. This is performed using sprintf(), to pad with 0 the interval to a 4-digit integer and then strptime() is used, telling to this function that the format is HHMM. This gives as a result a date-hour list of values, with the date of the execution used for the part day-year-month of the date. This is not really a problem, because I dont want the date part of the result, only the time (hour and minutes). Additionally, in the plot() function we add the xaxt = n parameter, in order to not show the x-axis, avoiding a wrong interpretation of the axis values. The correct data values for the axis are calculated using range() and applied to the plotted axis by axis.POSIXct(). The at parameter is a sequence from the first element to the last, in increments of 300, being this increments of 300 seconds (5min * 60s = 300s). This part of the procedure is much like the example in the help page for axis.POSIXct() function.

with(avgStepsByInterval, {
    plot(strptime(sprintf("%04d",as.numeric(avgStepsByInterval$interval)),"%H%M"), 
         avgStepsByInterval$avgsteps, 
         type="l", 
         xlab="5-minute interval", 
         ylab="Avg. steps taken", 
         main="Average daily activity pattern",
         xaxt = "n");
    axisInterval <- as.POSIXct(range(strptime(sprintf("%04d",as.numeric(avgStepsByInterval$interval)),"%H%M")));
    axis.POSIXct(1, at = seq(axisInterval[1], axisInterval[2], by = 300), format = "%H:%M");
});

3. Create a new dataset that is equal to the original dataset but with the missing data filled in:

The process starts making a copy of the original data into activityDataNAsFilled. Then, a copy of the data with NAs is made in activityDataNAs. Using the merge() function its possible to add each average value to its corresponding data row. The common column for the merge is the interval. The result is merged into activityDataNAsFilled and ordered using the arrange() function. The NAs values in steps are replaced by the average value and the extra column is deleted using subset().

activityDataNAsFilled <- activityData;
activityDataNAs <- activityData[is.na(activityData$steps),];
activityDataNAs <- merge(activityDataNAs,avgStepsByInterval);
activityDataNAsFilled <- arrange(merge(activityDataNAsFilled,activityDataNAs, all = TRUE), date, interval);
activityDataNAsFilled$steps[is.na(activityDataNAsFilled$steps)] <- 
    activityDataNAsFilled$avgsteps[is.na(activityDataNAsFilled$steps)];
activityDataNAsFilled <- subset(activityDataNAsFilled, select = c("steps","date","interval"));

Are there differences in activity patterns between weekdays and weekends?

For this part the weekdays() function may be of some help here. Use the dataset with the filled-in missing values for this part.

1. Create a new factor variable in the dataset with two levels weekday and weekend indicating whether a given date is a weekday or weekend day.

This is performed using the factor() function to encode the weekdays as an ordered factor. Then, the levels() function is used to sort each day into two categories: weekday or weekend.
